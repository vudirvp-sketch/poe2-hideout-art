# `img2hideout` — превращаем картинку в напольную композицию

Конвейер `img2hideout` превращает PNG/JPG в `.hideout` файл. Каждый
непрозрачный пиксель исходной картинки становится одним размещением
декорации. Цвет пикселя → ближайший цвет палитры → тип декорации.

## Базовый рецепт

```bash
hideout-art img2hideout picture.png -o art.hideout \
    --palette examples/palette.json \
    --width 120 --scale 2 \
    --hideout-name "Canal Hideout" --hideout-hash 60415 \
    --preview
```

`--preview` пишет рядом `art.preview.png` — точечную визуализацию того,
что попадёт в файл. Импортируйте в игру только когда превью устроит.

## Параметры, которые реально влияют на результат

### Размер: `--width`, `--scale`, `--step`

- `--width N` — ширина исходной картинки в пикселях после даунскейла.
  По умолчанию 120. Больше — детальнее, но экспоненциально больше
  размещений (и игра начинает тормозить).
- `--scale K` — мировых единиц на пиксель. По умолчанию 2. Больше —
  композиция физически крупнее.
- `--step N` — размещать декорацию каждые N пикселей. `--step 2` вдвое
  уменьшает количество размещений и **даёт каждой декорации место**.
  Полезно, когда в игре видны «дыры» между соседними размещениями.

**Типичная ошибка:** `--scale 2 --step 1` при широкой картинке —
декорации наслаиваются, образуя пиксельную кашу. Начинайте с
`--step 2 --scale 4` и уменьшайте, если нужно больше плотности.

### Прозрачность и фон: `--bg`, `--no-bg`, `--alpha-threshold`, `--no-alpha`

- Если у PNG есть alpha-канал — пиксели с `alpha < --alpha-threshold`
  автоматически пропускаются. Это правильный способ для transparency.
- Если PNG без alpha (например, JPG), `img2hideout` пропускает пиксели,
  близкие к цвету `--bg R G B` (по умолчанию чёрный) в радиусе
  `--bg-threshold` (по умолчанию 30).
- `--no-bg` отключает пропуск по цвету фона. `--no-alpha` форсит режим
  background-colour даже при наличии alpha-канала (legacy).

### Цветовая метрика: `--color-metric`

- `rgb` (по умолчанию) — обычное Евклидово расстояние в RGB.
- `weighted` — взвешивание по яркости (0.299R + 0.587G + 0.114B).
  Предотвращает, чтобы тёмно-зелёный пиксель ушёл к тёмно-коричневой
  декорации.
- `redmean` — perceptual метрика, хорошо работает в тёплом диапазоне
  (красный/оранжевый/коричневый).

### Дизеринг: `--dither`

Floyd-Steinberg error diffusion. Каждое квантование пикселя
распространяет ошибку на соседей — градиенты выглядят гораздо плавнее.
**Минус:** плотность размещений немного меняется — на границе двух
цветов появятся «шумные» одиночные пиксели.

### Джиттер: `--jitter`, `--jitter-seed`, `--jitter-variants`

Каждой декорации случайно назначается:
- поворот `r` — кратный 15° (24 значения по кругу);
- вариант `fv` — от 0 до `--jitter-variants - 1`.

По умолчанию все декорации стоят `r=0, fv=0` — «по линеечке». Для
травы/мха/песка это выглядит неестественно. `--jitter` включает
случайность, `--jitter-seed` делает её воспроизводимой.

### Ограничение области: `--bounds x_min,y_min,x_max,y_max`

Пропускает размещения, чьи мировые координаты выходят за прямоугольник.
Полезно, когда вы «очертили» играбельную зону убежища и не хотите, чтобы
декорации уходили в стены. Координаты — мировые, в тех же единицах, что
и `--origin-x` / `--origin-y`.

### Даунскейл: `--resample`

- `nearest` — пиксель-арт, сохраняет резкие края.
- `bilinear`, `bicubic` (по умолчанию) — сглаживание, для фото.
- `lanczos` — самое качественное сглаживание.

Для спрайтов и пиксель-арта используйте `nearest`. Для фото — `lanczos`.

## Как собрать хорошую палитру

1. **Определите «цветовые роли» субъекта.** Для 2B это: белый (платье),
   чёрный (повязка, волосы), серый (меч), телесный (кожа), розовый
   (свечение), красный (эмблема).
2. **Найдите в игре декорации нужных цветов.** Разместите по одной,
   экспортируйте `.hideout`, прогоните `hideout-art inspect` — в секции
   «Unknown hashes» будет хеш каждой новой декорации.
3. **Добавьте хеши** в `KNOWN_HASHES` (`src/hideout_art/constants.py`),
   а название — в `ART_TYPES` (если декорация чисто художественная).
4. **Соберите JSON-палитру.** См. `examples/palette.json` и
   `examples/palette_2b.json` (шаблон с TODO-плейсхолдерами).

### Минимальная палитра для портрета

4 цвета — абстракция. 6–8 цветов — узнаваемый силуэт. 10+ — узнаваемое
лицо. Чем меньше цветов в палитре, тем сильнее `--dither` улучшает
результат.

## Типичные проблемы

### «Видны дыры между декорациями»

Увеличьте `--step` или уменьшите `--scale`. Каждая декорация занимает
порядка 1 тайла (~23 мировых единиц в Canal Hideout). При `--scale 2`
две соседние декорации на расстоянии 2 единиц почти гарантированно
перекрываются — но в превью это не видно, потому что `render_png`
рисует их точками.

### «Градиент выглядит ступенчатым»

Включите `--dither`. Если палитра слишком узкая (4 цвета), добавьте
промежуточные.

### «Тёмно-зелёная трава попадает в коричневые волосы»

Переключитесь на `--color-metric weighted` или `redmean`.

### «Композиция выходит за границы убежища»

Задайте `--bounds x_min,y_min,x_max,y_max` в мировых координатах.
Размещения за пределами будут пропущены.

### «В превью всё окей, а в игре криво»

Превью — точечная визуализация, оно не знает реальный размер декораций.
Калибруйте: разместите в игре одну декорацию, экспортируйте,
посчитайте разницу координат между двумя соседними тайлами.
Полученное число = «мировых единиц на тайл».

## Что `img2hideout` НЕ умеет (пока)

- Multi-pass (сначала контур, потом заливка).
- Автоматический подбор размера под тайл.
- Анти-алиасинг по границам цветовых зон.
- Перенос палитры между убежищами (нужно вручную перепроверять хеши).

См. `STATUS.md` → «Что улучшать дальше».
