# `img2hideout` — превращаем картинку в напольную композицию

Конвейер `img2hideout` превращает PNG/JPG в `.hideout` файл. Каждый
непрозрачный пиксель исходной картинки становится одним размещением
декорации. Цвет пикселя → ближайший цвет палитры → тип декорации.

## Базовый рецепт

```bash
hideout-art img2hideout picture.png -o art.hideout \
    --palette examples/palette.json \
    --width 120 --scale 2 \
    --hideout-name "Canal Hideout" --hideout-hash 60415 \
    --preview
```

`--preview` пишет рядом `art.preview.png` — точечную визуализацию того,
что попадёт в файл. Импортируйте в игру только когда превью устроит.

## Параметры, которые реально влияют на результат

### Размер: `--width`, `--scale`, `--step`, `--tile-size`

- `--width N` — ширина исходной картинки в пикселях после даунскейла.
  По умолчанию 120. Больше — детальнее, но экспоненциально больше
  размещений (и игра начинает тормозить).
- `--scale K` — мировых единиц на пиксель. По умолчанию 2. Больше —
  композиция физически крупнее.
- `--step N` — размещать декорацию каждые N пикселей. `--step 2` вдвое
  уменьшает количество размещений и **даёт каждой декорации место**.
  Полезно, когда в игре видны «дыры» между соседними размещениями.
- `--tile-size N` — (новое в 0.2.1) физический размер декорации в
  мировых единицах. При передаче автоматически вычисляет
  `step = max(1, round(tile_size / scale))`, чтобы одно размещение
  приходилось на один тайл — без перекрытий и щелей. **Взаимно
  исключающее с `--step`.** Типичное значение: `--tile-size 23`
  (калибровано по пользовательским экспортам). Закрывает KI-1.

**Типичная ошибка:** `--scale 2 --step 1` при широкой картинке —
декорации наслаиваются, образуя пиксельную кашу. Начинайте с
`--tile-size 23 --scale 2` (или вручную `--step 12 --scale 2`)
и уменьшайте, если нужно больше плотности.

#### Калибровка `--tile-size`

Константа `DEFAULT_TILE_SIZE_WORLD_UNITS = 23` в
`src/hideout_art/constants.py` вычислена по дельтам между соседними
ручными размещениями в пользовательских экспортах (Maraket Rubble,
Coastal Pebble): соседние размещения сидят на расстоянии 15-23 мировых
единиц, что соответствует ~1 тайлу Canal Hideout. Если вы калибруете
под другое убежище — разместите две декорации в соседних тайлах,
экспортируйте `.hideout`, посчитайте разницу координат и передайте
полученное число в `--tile-size`.

### Прозрачность и фон: `--bg`, `--no-bg`, `--alpha-threshold`, `--no-alpha`

- Если у PNG есть alpha-канал — пиксели с `alpha < --alpha-threshold`
  автоматически пропускаются. Это правильный способ для transparency.
- Если PNG без alpha (например, JPG), `img2hideout` пропускает пиксели,
  близкие к цвету `--bg R G B` (по умолчанию чёрный) в радиусе
  `--bg-threshold` (по умолчанию 30).
- `--no-bg` отключает пропуск по цвету фона. `--no-alpha` форсит режим
  background-colour даже при наличии alpha-канала (legacy).

### Цветовая метрика: `--color-metric`

- `rgb` (по умолчанию) — обычное Евклидово расстояние в RGB.
- `weighted` — взвешивание по яркости (0.299R + 0.587G + 0.114B).
  Предотвращает, чтобы тёмно-зелёный пиксель ушёл к тёмно-коричневой
  декорации.
- `redmean` — perceptual метрика, хорошо работает в тёплом диапазоне
  (красный/оранжевый/коричневый).

### Дизеринг: `--dither`

Floyd-Steinberg error diffusion. Каждое квантование пикселя
распространяет ошибку на соседей — градиенты выглядят гораздо плавнее.
**Минус:** плотность размещений немного меняется — на границе двух
цветов появятся «шумные» одиночные пиксели.

### Джиттер: `--jitter`, `--jitter-seed`, `--jitter-variants`

Каждой декорации случайно назначается:
- поворот `r` — кратный 15° (24 значения по кругу);
- вариант `fv` — от 0 до `--jitter-variants - 1`.

По умолчанию все декорации стоят `r=0, fv=0` — «по линеечке». Для
травы/мха/песка это выглядит неестественно. `--jitter` включает
случайность, `--jitter-seed` делает её воспроизводимой.

### Ограничение области: `--bounds`

Пропускает размещения, чьи мировые координаты выходят за прямоугольник.
Полезно, когда вы «очертили» играбельную зону убежища и не хотите, чтобы
декорации уходили в стены. Координаты — мировые, в тех же единицах, что
и `--origin-x` / `--origin-y`.

Две формы:

- **Именованный шорткат:** `--bounds canal` (для Canal Hideout). Бокс
  `(700, 540, 860, 775)` откалиброван по пользовательской разметке из 11
  Cordilina-размещений по периметру играбельного полотна. Регистр не
  важен: `canal`, `CANAL`, `Canal` — все работают. См. `NAMED_BOUNDS` в
  `src/hideout_art/constants.py` — добавляйте новые убежища по мере
  появления экспортов.
- **Явный прямоугольник:** `--bounds x_min,y_min,x_max,y_max` — четыре
  целых через запятую.

```bash
# Canal Hideout — готовый шорткат
hideout-art img2hideout pic.png -o art.hideout \
    --palette examples/palette_warm.json \
    --tile-size 23 --width 80 --bounds canal --dither

# Другое убежище — пока только явно
hideout-art img2hideout pic.png -o art.hideout \
    --palette my_palette.json \
    --bounds 100,200,500,600
```

#### Калибровка `CANAL_HIDEOUT_BOUNDS`

Константа `CANAL_HIDEOUT_BOUNDS = (700, 540, 860, 775)` вычислена по
экспорту `исходники/Кордилой обозначил условные границы полотна...hideout`:
пользователь разместил 11 декораций «Cordilina» по периметру играбельного
полотна Canal Hideout и одну «Petrified Cave Figure» в условном центре.
Bounding box этих 11 Cordilina: x ∈ [707, 854], y ∈ [542, 772]. Округлено
до (700, 540, 860, 775) для CLI-эргономики; дельта <7 мировых единиц —
внутри пользовательской погрешности «условный центр плюс-минус».

Для других убежищ: разместите декорации по углам и сторонам
играбельной зоны, экспортируйте `.hideout`, посчитайте bounding box
через `hideout-art inspect`, добавьте запись в `NAMED_BOUNDS`.

### Даунскейл: `--resample`

- `nearest` — пиксель-арт, сохраняет резкие края.
- `bilinear`, `bicubic` (по умолчанию) — сглаживание, для фото.
- `lanczos` — самое качественное сглаживание.

Для спрайтов и пиксель-арта используйте `nearest`. Для фото — `lanczos`.

## Как собрать хорошую палитру

1. **Определите «цветовые роли» субъекта.** Для 2B это: белый (платье),
   чёрный (повязка, волосы), серый (меч), телесный (кожа), розовый
   (свечение), красный (эмблема).
2. **Найдите в игре декорации нужных цветов.** Разместите по одной,
   экспортируйте `.hideout`, прогоните `hideout-art inspect` — в секции
   «Unknown hashes» будет хеш каждой новой декорации.
3. **Добавьте хеши** в `KNOWN_HASHES` (`src/hideout_art/constants.py`),
   а название — в `ART_TYPES` (если декорация чисто художественная).
4. **Соберите JSON-палитру.** См. `examples/palette.json` и
   `examples/palette_2b.json` (шаблон с TODO-плейсхолдерами).

### Минимальная палитра для портрета

4 цвета — абстракция. 6–8 цветов — узнаваемый силуэт. 10+ — узнаваемое
лицо. Чем меньше цветов в палитре, тем сильнее `--dither` улучшает
результат.

## Типичные проблемы

### «Видны дыры между декорациями»

Увеличьте `--step` или уменьшите `--scale`. Каждая декорация занимает
порядка 1 тайла (~23 мировых единиц в Canal Hideout). При `--scale 2`
две соседние декорации на расстоянии 2 единиц почти гарантированно
перекрываются — но в превью это не видно, потому что `render_png`
рисует их точками.

### «Градиент выглядит ступенчатым»

Включите `--dither`. Если палитра слишком узкая (4 цвета), добавьте
промежуточные.

### «Тёмно-зелёная трава попадает в коричневые волосы»

Переключитесь на `--color-metric weighted` или `redmean`.

### «Композиция выходит за границы убежища»

Задайте `--bounds x_min,y_min,x_max,y_max` в мировых координатах.
Размещения за пределами будут пропущены.

### «В превью всё окей, а в игре криво»

Превью — точечная визуализация, оно не знает реальный размер декораций.
Калибруйте: разместите в игре одну декорацию, экспортируйте,
посчитайте разницу координат между двумя соседними тайлами.
Полученное число = «мировых единиц на тайл».

## Что `img2hideout` НЕ умеет (пока)

- Multi-pass (сначала контур, потом заливка).
- Анти-алиасинг по границам цветовых зон.
- Перенос палитры между убежищами (нужно вручную перепроверять хеши).
- Использование `DECORATION_FOOTPRINT_CATALOG` для запрета размещения
  крупных декораций рядом с соседями (каталог существует с 0.2.3, но
  img2hideout его пока не использует — это часть будущего multi-pass).

См. `STATUS.md` → «Что улучшать дальше».

## Каталог размеров декораций (0.2.3)

`DECORATION_FOOTPRINT_CATALOG` в `src/hideout_art/constants.py` хранит
оценку **placement footprint** для каждой из 27 art-декораций:

- `samples` — сколько размещений этой декорации есть в `исходники/*.hideout`.
- `min_spacing_wu` — минимальная наблюдаемая дистанция между двумя
  same-hash размещениями (верхняя граница placement-тайла).
- `median_spacing_wu` — типичная дистанция (может быть завышена, если
  пользователь разбрасывал декорации по всему полотну — например,
  Cordilina по периметру Canal Hideout даёт median=144.8 wu).
- `estimated_tile_footprint` — `ceil(min_spacing_wu / 23 * 2) / 2`
  (округление вверх до 0.5 тайла, чтобы не занизить размер).
- `confidence` — `high` (≥5 образцов), `medium` (3-4), `low` (2),
  `single` (1, замер невозможен), `none` (декорации нет в исходниках).

### Что каталог измеряет, а что нет (KI-10)

Каталог измеряет **placement footprint** — размер «клетки», в которой
декорация размещается на сетке. Это то, что нужно для `img2hideout`
step-калибровки.

Каталог **НЕ** измеряет **sprite bounds** — реальные визуальные
габариты декорации. Видимая крона дерева выходит за пределы placement-
тайла; повёрнутые размещения могут иметь другие эффективные границы.
Эти данные невозможно получить из `.hideout` файлов — только из
внутриигровых замеров или VLM-анализа скриншотов.

### Доверие к low-confidence записям

Записи с `confidence=low` (2 образца) — шумные. Единственная пара
размещений могла оказаться далеко друг от друга по совпадению, а не
потому что декорация реально такого размера. Пример: `Marble Table`
показывает `min=45.6 wu → 2.0 tile`, но это `low` — скорее всего,
просто совпадение, и стол на самом деле 1 тайл. Не полагайтесь на
low-confidence значения без дополнительной проверки.

### Что исключено

Atziri Statue и другие функциональные объекты (Stash, Waypoint, NPC и
т.д.) намеренно исключены из каталога — по запросу пользователя они
слишком крупные и в художественных композициях не участвуют.

### Как обновить каталог

1. Добавьте новые размещения в `исходники/*.hideout` (или пришлите
   новый экспорт).
2. Запустите `python scripts/measure_decorations.py` — увидите
   таблицу и JSON-дамп.
3. Скопируйте новые значения в `DECORATION_FOOTPRINT_CATALOG` в
   `src/hideout_art/constants.py`.
4. Запустите `pytest tests/test_footprints.py` — тест
   `test_sample_counts_match_real_exports` упадёт, если `samples` в
   каталоге не совпадает с реальным числом размещений в исходниках.

## Доступные палитры

- `examples/palette.json` — базовая 4-цветная палитра Canal Hideout
  (розовый/зелёный/светло-зелёный/тёмно-коричневый). Для сердечек и
  абстрактных силуэтов.
- `examples/palette_warm.json` — 9-цветная тёплая палитра (4 исходных +
  5 Maraket/Coastal Pebble). **Загружается и работает.** Для пустынь,
  дерева, камня, осенних сцен.
- `examples/palette_2b.json` — шаблон для холодных портретов (2B-style).
  В 0.2.4 **3 из 6 TODO заполнены** Marble-серией (VLM-measured RGB):
  - `white`  → Marble Fountain (230, 230, 220)
  - `silver` → Marble Table    (200, 200, 195)
  - `gray`   → Marble Bench    (210, 210, 205)
  - `gray` alt → Marble Walls  (210, 210, 205)
  
  Остаются 3 TODO: `black` (повязка/волосы), `skin` (кожа), `red`
  (эмблема) — ни одна известная декорация не подходит. Палитра пока
  НЕ загружается через `Palette.from_json_file` из-за оставшихся TODO_*.
  Если их временно вырезать — загрузится и работает.

### VLM-измеренные RGB (0.2.4)

Источником RGB-значений для Marble-серии и Cave Fossil/Coral/Brazier
послужил VLM-анализ скриншота `исходники/еще элементы.jpg` через
`glm-4.6v`. VLM идентифицировал каждую декорацию визуально и сообщил
mid-tone RGB.

**Важная находка:** Cave Fossil — **коричневый** (140, 110, 80), а не
светло-серый/белый, как предполагалось в 0.2.2. Это значит, что Cave
Fossil **не** подходит для холодной 2B-палитры. Cave Coral и Summit
Brazier тоже тёплые. Только Marble-серия оказалась действительно cool.

Полная таблица измеренных значений — в `examples/palette_2b.json` →
`_0_2_4_measured_rgb_summary`, с указанием метода и источника.
