## 1. Как читать каталог

| Колонка | Что значит |
|---|---|
| **Имя** | Имя декора в локализованной версии (русский). Внутри .hideout используется именно оно. |
| **Hash** | uint32, стабильный идентификатор. Языко-независимый — меняется только имя, hash тот же. |
| **fv-диапазон** | Варианты декора, наблюдаемые в `исходники/`. `fv` = `(flip_x << 7) \| variant`. |
| **Размер (tile)** | Оценка footprint'а в тайлах (1 тайл ≈ 23 wu). От `scripts/measure_decorations.py`. |
| **Цвет (RGB)** | Pixel-sampled из игры (если есть) или оценка по JPG. |
| **Назначение** | Чем обычно полезен при рисовании (fill/stroke/accent/single). |
| **Источник** | В каком файле `исходники/` встречается. |

## 4. Геометрия «Убежища в каналах»

- `hideout_hash`: 60415
- Игровое поле (canvas) — bounding box всех artistic placement'ов в
  `исходники/` ≈ **x ∈ [707, 854], y ∈ [541, 772]**.
- Размер canvas: ~147 × 231 world units.
- Y растёт вверх (см. `docs/format.md`).
- Functional singletons стоят по периметру (y ≈ 519-786, x ≈ 660-855).
- Центр для рисунков: ~ (770, 660).

---

## 5. Поле `r` (rotation)

PoE2 хранит вращение как uint16: `degrees = r / 65536 * 360`.

- `r = 0` — 0°
- `r = 16384` — 90°
- `r = 32768` — 180°
- `r = 49152` — 270°

PoE2 editor snap'ит к 15° (значения 0, 2730, 5461, 8192, 10922, 13653, 16384, ...),
но поле **не ограничено** — можно любое значение.

**Декор можно вращать вокруг своей оси.** Это полезно, когда декор
несимметричный (Бревно, Жаровня, Буковое дерево, Кустарники) —
поворот даёт визуальное разнообразие. Симметричные декоры (Камни, Галька)
вращать нет смысла — форма та же.

## 6. Поле `fv` (flip + variant)

`fv = (flip_x << 7) | variant`:

- `flip_x = 0/1` — отразить декор по горизонтали (бит 7 = 0x80).
- `variant = 0..127` — индекс варианта модели (младшие 7 бит).

В `исходники/`:
- `flip_x` всегда `0` (отражений нет).
- `variant` варьируется в диапазонах, указанных в колонке «fv-диапазон»
  выше. Например, для «Маракетских обломков» — `0..10`, для
  «Береговой гальки» — `0..8`.

> Если в каталоге указано `0..8` — это значит, что варианты с `0` по `8`
> точно существуют в игре. Варианты за пределами диапазона **могут**
> существовать, но не подтверждены.

## 7. Анти-паттерны (что НЕ делать)

| Анти-паттерн | Почему плохо |
|---|---|
| `json.load()` для чтения .hideout | Теряет duplicate keys → счёт 1 вместо 9. |
| `json.dump()` для записи .hideout | Теряет duplicate keys → сломанный файл. |
| Придумывать hash декора | Игра не распознает, объект не появится. |
| Использовать декор не из «Убежища в каналах» | Hash может быть тем же, но hideout_hash другой → не валидируется. |
| Ставить декор на координаты functional singletons | Будут наезжать друг на друга. |
| Игнорировать fv-диапазон | Несуществующий variant → декор не появится. |

## 8. Что требует уточнения (Known Issues)

- **KI-12**: RGB marble-декоров (Мраморная скамья / стены / фонтан) —
  pixel-sampling показал тёмные значения (76-110), но визуально декор
  светлый. Возможно, sampling попал в тень под объектом. Нужен ручной
  скриншот.
- **KI-13**: Размер (footprint) для single-observation декоров (1
  placement в `исходники/`) — не измерен. Поставить 2+ placement'а и
  прислать новый `.hideout`.
- **KI-16 (RESOLVED в v0.6.2)**: RGB для Falling Sand и Long Grass
  исправлены на VISUAL значения (розовый / зелёный). Pixel-sampling
  0.2.6 был неверным (попал в тени/промежутки между частицами).
  Подтверждено скриншотом пользователя 234156.jpg.
- **KI-17**: Fringe Moss (Мох с опушки, hash=1459723677) — 0 placements
  в `исходники/`, hash не верифицирован. Пользователь использует этот
  декор (виден на скриншоте 234156.jpg), но нужен `.hideout` файл для
  подтверждения hash.
