# STATUS.md

Текущее состояние проекта `poe2-hideout-art`. Короткая живая памятка для
агента/человека: что работает, что сломано, что делать дальше. Длинную
историю изменений не ведём — для этого есть `CHANGELOG.md`.

## Что работает (проверено 2026-07-09, версия 0.2.2)

- Парсер `.hideout` (толерантный, регэксп, сохраняет дубликаты ключей).
- Врайтер (байт-совместимый, дубликаты ключей сохраняет).
- Геометрические трансформации: `shift`, `rotate`, `mirror_x`, `mirror_y`,
  `recombine`.
- Превью в PNG (matplotlib, по одной точке на размещение).
- Конвейер `img2hideout`:
  - базовый режим (4-цветная палитра, без dither, без alpha) совместим
    с 0.1.0 побайтово;
  - alpha-канал PNG — пиксели с alpha < порога пропускаются автоматически;
  - 3 цветовых метрики: `rgb`, `weighted`, `redmean`;
  - Floyd-Steinberg dithering (опционально);
  - jitter `r` (кратный 15°) и `variant` (опционально, с seed);
  - `step` ИЛИ `tile_size` — размещение каждые N пикселей / один тайл;
  - `bounds` — отсечение по мировому прямоугольнику **или** именованный
    шорткат (`--bounds canal` для Canal Hideout);
  - `resample` — nearest/bilinear/bicubic/lanczos;
  - `--preview` в CLI — превью PNG рядом с выходным файлом.
- CLI: `inspect`, `layers`, `stats`, `preview`, `shift`, `transfer`,
  `img2hideout`.
- Каталог хешей: **46 известных декораций** (11 функциональных + 8 NPC +
  4 исходных art + 5 тёплых Maraket + 18 новых Canal из «исходники/»).
- Геометрия Canal Hideout: `CANAL_HIDEOUT_HASH = 60415`,
  `CANAL_HIDEOUT_BOUNDS = (700, 540, 860, 775)` — bounding box играбельного
  полотна, вычисленный по 11 Cordilina-размещениям пользовательской
  разметки границ.
- Тесты: **140 pytest-тестов** (139 проходят, 1 skipped — требует старую
  папку `upload/` с экспортом «Береговая галька», которого нет в репе).

### KI-2 (обновлено в 0.2.2)

В 0.2.2 добавлены 18 новых декораций Canal Hideout, включая **4 marble**
серии (Bench/Table/Walls/Fountain) и **Cave Fossil** — кандидаты на
«белый/светло-серый» для холодных палитр. Точные RGB пока не измерены;
пользователь должен сделать скриншоты в игре и присылать — добавлю в
`palette_2b.json` и закрою KI-2 полностью.

## Что улучшать дальше (не в этой итерации)

- **Multi-pass: outline + fill.** Сначала рисуем контур, потом заливку —
  узнаваемость фигур сильно вырастет. Реализуется в `img2hideout.py`
  через дополнительный параметр `outline_color` в палитре +
  двухпроходной рендер.
- **Каталог размеров декораций.** Знать, что «Long Grass ≈ 1 тайл»,
  «Atziri Statue ≈ 4×4 тайла», «Beech Tree ≈ 2×2 тайла», и не размещать
  крупную декорацию там, где она перекроет соседей. Пока — только
  усреднённый `DEFAULT_TILE_SIZE_WORLD_UNITS = 23`. В 0.2.2 появились
  крупные декорации (`Petrified Cave Figure`, `Beech Tree`), для которых
  это особенно важно.
- **Точные RGB для холодных декораций.** Marble-серия и Cave Fossil —
  кандидаты на white/light-gray для `palette_2b.json`. Пользователь
  присылает скриншот — измеряю median RGB и заполняю TODOs.
- **Поддержка нескольких убежищ.** Сейчас `NAMED_BOUNDS` содержит только
  `canal`. Добавить новые записи по мере появления экспортов других
  убежищ (с пользовательской разметкой границ по образцу).
- **SVG / Lua экспорт.** Для пользователей, которые хотят ручной доводки.

## Базовые принципы при правках

1. **Сначала в STATUS.md, потом фикс.** Новый баг → новый KI-N → потом код.
2. **Никогда не ломать существующие тесты.** Все новые опции — opt-in,
   дефолты сохраняют текущее поведение.
3. **Никогда не выдумывать хеши.** Нет в `KNOWN_HASHES` → помечаем как
   unknown, не подменяем.
4. **Документация короткая.** Этот файл — не больше 100 строк.
