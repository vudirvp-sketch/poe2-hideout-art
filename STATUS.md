# STATUS.md

Текущее состояние проекта `poe2-hideout-art`. Этот файл — короткая живая
памятка для агента/человека: что работает, что сломано, что делать дальше.
Длинную историю изменений не ведём — для этого есть `CHANGELOG.md`.

## Что работает (проверено 2026-07-09)

- Парсер `.hideout` (толерантный, регэксп, сохраняет дубликаты ключей).
- Врайтер (байт-совместимый, дубликаты ключей сохраняет).
- Геометрические трансформации: `shift`, `rotate`, `mirror_x`, `mirror_y`,
  `recombine`.
- Превью в PNG (matplotlib, по одной точке на размещение).
- Конвейер `img2hideout` (расширен в 0.2.0):
  - базовый режим (4-цветная палитра, без dither, без alpha) совместим
    с 0.1.0 побайтово;
  - alpha-канал PNG — пиксели с alpha < порога пропускаются автоматически;
  - 3 цветовых метрики: `rgb`, `weighted`, `redmean`;
  - Floyd-Steinberg dithering (опционально);
  - jitter `r` (кратный 15°) и `variant` (опционально, с seed);
  - `step` — размещение каждые N пикселей по x/y;
  - `bounds` — отсечение по мировому прямоугольнику;
  - `resample` — nearest/bilinear/bicubic/lanczos;
  - `--preview` в CLI — превью PNG рядом с выходным файлом.
- CLI: `inspect`, `layers`, `stats`, `preview`, `shift`, `transfer`,
  `img2hideout`.
- Каталог хешей: 23 известных декорации из Canal Hideout.
- Тесты: 44 pytest-теста (33 прежних + 11 новых для `img2hideout`).

## Известные проблемы (Known Issues)

### KI-1: `img2hideout` даёт заметные «дыры» в композиции

**Симптом.** На реальном рендере в игре видно «пиксельные» пустоты между
соседними размещениями. На скриншоте пользователя это особенно заметно на
зелёном контуре сердца.

**Причина.** Дефолтный `scale=2` слишком мелкий — 2 мировых единицы на
пиксель, а тайл в PoE2 занимает ~23 мировых единиц. Декорации физически
не перекрывают друг друга, и между ними остаются видимые щели.

**Решение (частичное, в 0.2.0).** Добавлены параметры `tile_size`-style
через `step` — пользователь может явно задать «1 размещение на N
пикселей» вместо абстрактного `scale`. `--step 2` рекомендуется как
стартовая точка.

**Что ещё нужно.** Реальный размер каждой декорации в тайлах известен
только в игре. Без калибровки по скриншоту идеальную плотность не получить.

### KI-2: Нет white/gray/black декораций в палитре

**Симптом.** Текущая палитра (`examples/palette.json`) содержит только 4
цвета: розовый, зелёный, светло-зелёный, тёмно-коричневый. Этого хватает
для сердечка или простого силуэта, но **недостаточно для портрета** —
например, 2B из NieR требует белого (платье), чёрного (повязка, волосы),
серого (меч).

**Решение (в 0.2.0).** В `examples/palette_2b.json` лежит рекомендуемый
шаблон палитры для портретов типа 2B с TODO-плейсхолдерами и инструкцией
по заполнению. Пользователь должен найти в игре белые/чёрные/серые
декорации, разместить по одной, экспортировать `.hideout`, прогнать
`hideout-art inspect` — и вписать хеш в `KNOWN_HASHES`.

### KI-3 → исправлен в 0.2.0

Добавлены опции `color_metric`: `"rgb"` (по умолчанию), `"weighted"`,
`"redmean"`. См. `docs/img2hideout.md`.

### KI-4 → исправлен в 0.2.0

Добавлен Floyd-Steinberg dithering (`dither=True` или `--dither`).

### KI-5 → исправлен в 0.2.0

PNG с alpha-каналом обрабатываются корректно — пиксели с alpha < порога
пропускаются автоматически.

### KI-6 → исправлен в 0.2.0

Добавлен параметр `jitter` (и `--jitter` в CLI) — случайно выбирает
вариант (0..N-1) и поворот (кратный 15°) для каждого размещения.

### KI-7 → исправлен в 0.2.0

Добавлен параметр `bounds` (и `--bounds` в CLI) — отсечение по мировому
прямоугольнику. Пользователь может «очертить» границы убежища и задать
их явно.

### KI-8 → исправлен в 0.2.0

Добавлен флаг `--preview` в CLI `img2hideout` — генерирует PNG с превью
рядом с выходным файлом.

## Что улучшать дальше (не в этой итерации)

- **Калибровка tile_size по скриншоту.** Пользователь присылает скриншот
  с известными размещениями → мы вычисляем world_units_per_tile.
- **Multi-pass: outline + fill.** Сначала рисуем контур, потом заливку —
  узнаваемость фигур сильно вырастет.
- **Каталог размеров декораций.** Знать, что «Long Grass ≈ 1 тайл»,
  «Atziri Statue ≈ 4×4 тайла», и не размещать крупную декорацию там,
  где она перекроет соседей.
- **Поддержка нескольких убежищ.** Сейчас палитра одна — для Canal Hideout.
  Завести отдельные палитры для каждого убежища.
- **SVG / Lua экспорт.** Для пользователей, которые хотят ручной доводки.

## Базовые принципы при правках

1. **Сначала в STATUS.md, потом фикс.** Новый баг → новый KI-N → потом код.
2. **Никогда не ломать существующие тесты.** Все новые опции — opt-in,
   дефолты сохраняют текущее поведение.
3. **Никогда не выдумывать хеши.** Нет в `KNOWN_HASHES` → помечаем как
   unknown, не подменяем.
4. **Документация короткая.** Этот файл — не больше 200 строк.
