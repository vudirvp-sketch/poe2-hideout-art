# STATUS.md

Текущее состояние проекта `poe2-hideout-art`. Короткая живая памятка для
агента/человека: что работает, что сломано, что делать дальше. Длинную
историю изменений не ведём — для этого есть `CHANGELOG.md`.

## Что работает (проверено 2026-07-09, версия 0.2.1)

- Парсер `.hideout` (толерантный, регэксп, сохраняет дубликаты ключей).
- Врайтер (байт-совместимый, дубликаты ключей сохраняет).
- Геометрические трансформации: `shift`, `rotate`, `mirror_x`, `mirror_y`,
  `recombine`.
- Превью в PNG (matplotlib, по одной точке на размещение).
- Конвейер `img2hideout`:
  - базовый режим (4-цветная палитра, без dither, без alpha) совместим
    с 0.1.0 побайтово;
  - alpha-канал PNG — пиксели с alpha < порога пропускаются автоматически;
  - 3 цветовых метрики: `rgb`, `weighted`, `redmean`;
  - Floyd-Steinberg dithering (опционально);
  - jitter `r` (кратный 15°) и `variant` (опционально, с seed);
  - `step` ИЛИ `tile_size` — размещение каждые N пикселей / один тайл;
  - `bounds` — отсечение по мировому прямоугольнику;
  - `resample` — nearest/bilinear/bicubic/lanczos;
  - `--preview` в CLI — превью PNG рядом с выходным файлом.
- CLI: `inspect`, `layers`, `stats`, `preview`, `shift`, `transfer`,
  `img2hideout`.
- Каталог хешей: **28 известных декораций** (23 из Canal Hideout +
  5 новых Maraket/Coastal Pebble от пользователя).
- Тесты: **70 pytest-тестов** (44 прежних + 26 новых для 0.2.1).

## Известные проблемы (Known Issues)

### KI-1 → исправлен в 0.2.1

Добавлен параметр `tile_size` (и `--tile-size` в CLI). При передаче
`--tile-size 23` система автоматически вычисляет
`step = max(1, round(tile_size / scale))`, чтобы одно размещение
приходилось на один тайл (без перекрытий и щелей). Константа
`DEFAULT_TILE_SIZE_WORLD_UNITS = 23` вычислена по дельтам между
соседними размещениями в пользовательских экспортах (Maraket Rubble,
Coastal Pebble): соседние размещения сидят на расстоянии 15-23 мировых
единиц, что соответствует ~1 тайлу. `step` и `tile_size` — взаимно
исключающие.

### KI-2 → частично исправлен в 0.2.1

Из пользовательских экспортов добавлены 5 новых декораций:
Maraket Rubble, Maraket Treasures, Maraket Samovar, Maraket Ornament,
Coastal Pebble — все **тёплых земляных тонов** (tan/khaki/light gray).
Создана `examples/palette_warm.json` — рабочая 9-цветная палитра для
тёплых сцен (пустыни, дерево, камень, осень). `palette_2b.json`
остаётся шаблоном — для холодных тонов (белый/чёрный/нейтральный серый)
TODO-плейсхолдеры ещё нужно заполнить, найдя соответствующие декорации
в игре.

### KI-9 (новая) → исправлена в 0.2.1

**Баг:** `Placement.is_art` использовал `self.name in ART_TYPES`, но
`.hideout` файлы локализованы (русские имена типа «Береговая галька»),
а `ART_TYPES` хранит английские канонические имена. В результате для
русскоязычных экспортов `is_art` всегда возвращал `False`, и фильтры
`art_only=True` теряли все размещения.

**Фикс:** `is_art` теперь резолвит через `HASH_TO_NAME.get(self.hash)`
и только потом смотрит в `ART_TYPES`. Если хеш неизвестен — fallback
на исходное имя (legacy behavior). Тесты: `test_is_art_resolves_via_hash_not_name`,
`test_is_art_false_for_unknown_hash`, `test_is_art_false_for_known_functional_object`.

### KI-3..KI-8 → исправлены в 0.2.0

См. `CHANGELOG.md`. Кратко: `color_metric` (KI-3), `dither` (KI-4),
alpha-channel (KI-5), `jitter` (KI-6), `bounds` (KI-7), `--preview` (KI-8).

## Что улучшать дальше (не в этой итерации)

- **Multi-pass: outline + fill.** Сначала рисуем контур, потом заливку —
  узнаваемость фигур сильно вырастет. Реализуется в `img2hideout.py`
  через дополнительный параметр `outline_color` в палитре +
  двухпроходной рендер.
- **Каталог размеров декораций.** Знать, что «Long Grass ≈ 1 тайл»,
  «Atziri Statue ≈ 4×4 тайла», и не размещать крупную декорацию там,
  где она перекроет соседей. Пока — только усреднённый
  `DEFAULT_TILE_SIZE_WORLD_UNITS = 23`.
- **Холодные декорации для palette_2b.json.** Пользователь должен найти
  в игре белые/чёрные/нейтрально-серые декорации, разместить по одной,
  экспортировать `.hideout`, прислать файл — добавим хеши в `KNOWN_HASHES`.
- **Поддержка нескольких убежищ.** Сейчас палитра одна — для Canal Hideout.
  Завести отдельные палитры для каждого убежища.
- **SVG / Lua экспорт.** Для пользователей, которые хотят ручной доводки.

## Базовые принципы при правках

1. **Сначала в STATUS.md, потом фикс.** Новый баг → новый KI-N → потом код.
2. **Никогда не ломать существующие тесты.** Все новые опции — opt-in,
   дефолты сохраняют текущее поведение.
3. **Никогда не выдумывать хеши.** Нет в `KNOWN_HASHES` → помечаем как
   unknown, не подменяем.
4. **Документация короткая.** Этот файл — не больше 150 строк.
